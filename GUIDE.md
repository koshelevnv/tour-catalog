## Управление Docker

Все команды выполняются на сервере (SSH или терминал VM):

```bash
cd /home/user/tour-catalog

docker compose up -d          # запустить всё
docker compose ps             # статус контейнеров
docker compose down           # остановить
docker compose restart frontend   # перезапустить после изменений vite.config.js
docker compose restart backend    # перезапустить после изменений .env или config/
```

## Сайт (http://192.168.2.133:5173)

### Переключение темы

В правом углу шапки — кнопка со значком солнца или луны. Переключает светлую и тёмную тему. Предпочтение сохраняется в браузере (localStorage), по умолчанию — тёмная тема.

### Главная страница — каталог

Открой http://192.168.2.133:5173 — каталог туров.

- **Категории** — горизонтальный ряд чипов (Горные, Морские, Семейные и др.); клик фильтрует список немедленно
- **Фильтры** — панель под категориями: слайдеры диапазона «Длительность» и «Цена» (границы берутся из реальных данных) + Дата отправления
- **Режим загрузки** — настраивается в админке: бесконечная прокрутка / кнопка «Загрузить ещё» / пагинация
- Строка «Найдено туров: N» (слева) + выпадающий список **Сортировка** (справа): Сначала новые / Сначала старые / Сначала дешевые / Сначала дорогие / Длительность меньше/больше / По алфавиту А→Я/Я→А

### Страница каталога /tours

Альтернативный каталог с боковой панелью фильтров и постраничной пагинацией. Над сеткой туров — строка с количеством и сортировкой.

### Страница тура (/tours/slug)

Клик по карточке тура → детальная страница:

- **Фотогалерея** — клик по фото открывает лайтбокс (стрелки ← →, Esc для закрытия)
- **Варианты** — таблица дат и цен справа
- **Карта маршрута** — Yandex Maps с пронумерованными точками и маршрутом
  - Переключатель режимов (Пешком / Авто / Прямая) в правом нижнем углу карты
  - При недоступности маршрута — автоматический fallback с уведомлением

### Поиск по турам

В шапке сайта — поле «Куда поедем?»:

1. Начни вводить запрос (от 2 символов) — появится выпадающий список подсказок
   - Клик по туру в подсказках — откроет его страницу
   - Клик «Показать все результаты» — полный поиск
2. Нажми Enter или кнопку поиска для семантического поиска

Поиск объединяет векторное сходство (pgvector) и текстовое совпадение (ILIKE) — тур гарантированно найдётся в обоих случаях.

## Админ-панель (http://192.168.2.133:5173/admin)

### Вход

```
http://192.168.2.133:5173/admin/login

Email:    admin@example.com
Пароль:   admin_secret
```

### Список туров (/admin/tours)

Таблица всех туров: название, тип, дни, цена от.

- **Edit** — редактировать тур
- **Delete** — удалить тур (диалог подтверждения)
- **«+ Создать тур»** — создать новый тур

### Создание/редактирование тура

#### Генерация через ИИ (необязательно)

1. Выбери чекбоксами, **какие блоки** заполнить нейросетью (по умолчанию — только Описание)
2. Введи запрос: `Тур по Байкалу, 5 дней, экологический`
3. Нажми **«Сгенерировать»** — отмеченные поля заполнятся автоматически

> Провайдер и ключ настраиваются в `/admin/settings` → блок «ИИ / LLM»

#### Основные поля

Название (обязательное), тип тура, описание (форматированный редактор Tiptap).

#### Фотографии

Нажми «Выбрать фото» → загрузи изображение. Фотографии можно **перетаскивать** для изменения порядка — первое фото становится обложкой.

#### Варианты (даты и цены)

«+ Добавить» — новая строка с диапазоном дат и ценой.

#### Точки маршрута (карта)

Кликни на карту — добавится точка с номером. Введи метку (необязательно).

### Настройки (/admin/settings)

Вкладки: Интеграции, Сайт, Типы туров, Аккаунт, Тексты интерфейса.

- **Яндекс.Карты** — ключ API
- **ИИ / LLM** — Anthropic или OpenRouter, ключ и модель
- **Сайт** — SEO-теги, режим загрузки каталога (infinite / load_more / pagination), количество на странице
- **Аккаунт** — смена email и пароля администратора
- **Тексты интерфейса** — переопределение любых UI-строк без редактирования кода

## Управление данными

### Сброс базы

```bash
docker exec tour-catalog-backend-1 php artisan migrate:fresh --seed
```

Удаляет все данные и создаёт тестовые туры.

### Генерация embeddings

```bash
docker exec tour-catalog-backend-1 php artisan tours:generate-embeddings
```

Запусти если добавил туры вручную и хочешь, чтобы они появлялись в семантическом поиске.

### Просмотр логов

```bash
docker compose logs -f            # все сервисы
docker compose logs -f backend    # только backend
docker compose logs -f frontend   # только frontend
```

## API endpoints (справочник)

### Публичные

| Метод | URL | Описание |
|---|---|---|
| GET | /api/tour-types | Список типов туров |
| GET | /api/tours | Каталог (type, duration_min/max, price_min/max, date_from/to, page) |
| GET | /api/tours/meta | Мин/макс длительности и цены (для слайдеров фильтра) |
| GET | /api/tours/suggest?q=... | Быстрые подсказки autocomplete |
| GET | /api/tours/search?q=... | Семантический + текстовый поиск |
| GET | /api/tours/{slug} | Детальная страница тура |
| GET | /api/settings | Публичные настройки (Yandex key, display settings) |
